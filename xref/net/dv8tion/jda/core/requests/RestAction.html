35  /**
36   * A class representing a terminal between the user and the discord API. <br>
37   * This is used to offer users the ability to decide how JDA should limit a Request.
38   *
39   * <p>Methods that return an instance of RestAction require an additional step to complete the
40   * execution. Thus the user needs to append a follow-up method.
41   *
42   * <p>A default RestAction is issued with the following operations:
43   *
44   * <ul>
45   *   <li>{@link #queue()}, {@link #queue(Consumer)}, {@link #queue(Consumer, Consumer)} <br>
46   *       The fastest and most simplistic way to execute a RestAction is to queue it. <br>
47   *       This method has two optional callback functions, one with the generic type and another with
48   *       a failure exception.
49   *   <li>{@link #submit()}, {@link #submit(boolean)} <br>
50   *       Provides a Future representing the pending request. <br>
51   *       An optional parameter of type boolean can be passed to disable automated rate limit
52   *       handling. (not recommended)
53   *   <li>{@link #complete()}, {@link #complete(boolean)} <br>
54   *       Blocking execution building up on {@link #submit()}. <br>
55   *       This will simply block the thread and return the Request result, or throw an exception.
56   *       <br>
57   *       An optional parameter of type boolean can be passed to disable automated rate limit
58   *       handling. (not recommended)
59   * </ul>
60   *
61   * The most efficient way to use a RestAction is by using the asynchronous {@link #queue()}
62   * operations. <br>
63   * These allow users to provide success and failure callbacks which will be called at a convenient
64   * time.
65   *
66   * <h2>Planning Execution</h2>
67   *
68   * To <u>schedule</u> a RestAction we provide both {@link #queue()} and {@link #complete()} versions
69   * that will be executed by a {@link java.util.concurrent.ScheduledExecutorService
70   * ScheduledExecutorService} after a specified delay:
71   *
72   * <ul>
73   *   <li>{@link #queueAfter(long, TimeUnit)} <br>
74   *       Schedules a call to {@link #queue()} with default callback {@link
75   *       java.util.function.Consumer Consumers} to be executed after the specified {@code delay}.
76   *       <br>
77   *       The {@link java.util.concurrent.TimeUnit TimeUnit} is used to convert the provided long
78   *       into a delay time. <br>
79   *       Example: {@code queueAfter(1, TimeUnit.SECONDS);} <br>
80   *       will call {@link #queue()} <b>1 second</b> later.
81   *   <li>{@link #submitAfter(long, TimeUnit)} <br>
82   *       This returns a {@link java.util.concurrent.ScheduledFuture ScheduledFuture} which can be
83   *       joined into the current Thread using {@link java.util.concurrent.ScheduledFuture#get()}
84   *       <br>
85   *       The blocking call to {@code submitAfter(delay, unit).get()} will return the value processed
86   *       by a call to {@link #complete()}
87   *   <li>{@link #completeAfter(long, TimeUnit)} <br>
88   *       This operation simply sleeps for the given delay and will call {@link #complete()} once
89   *       finished sleeping.
90   * </ul>
91   *
92   * <p>All of those operations provide overloads for optional parameters such as a custom {@link
93   * java.util.concurrent.ScheduledExecutorService ScheduledExecutorService} instead of using the
94   * default global JDA executor. Specifically {@link #queueAfter(long, TimeUnit)} has overloads to
95   * provide a success and/or failure callback due to the returned {@link
96   * java.util.concurrent.ScheduledFuture ScheduledFuture} not being able to provide the response
97   * values of the {@link #queue()} callbacks.
98   *
99   * <h1>Using RestActions</h1>
100  *
101  * The most common way to use a RestAction is not using the returned value. <br>
102  * For instance sending messages usually means you will not require to view the message once it was
103  * sent. Thus we can simply use the <b>asynchronous</b> {@link #queue()} operation which will be
104  * executed on a rate limit worker thread in the background, without blocking your current thread:
105  *
106  * <pre><code>
107  *      {@link net.dv8tion.jda.core.entities.MessageChannel MessageChannel} channel = event.getChannel();
108  *     {@literal RestAction<Message>} action = channel.sendMessage("Hello World");
109  *      action.{@link #queue() queue()}; // Execute the rest action asynchronously
110  * </code></pre>
111  *
112  * <p>Sometimes it is important to access the response value, possibly to modify it later. <br>
113  * Now we have two options to actually access the response value, either using an asynchronous
114  * callback {@link java.util.function.Consumer Consumer} or the (not recommended) {@link
115  * #complete()} which will block the current thread until the response has been processed and joins
116  * with the current thread.
117  *
118  * <h2>Example Queue: (recommended)</h2>
119  *
120  * <pre><code>
121  *     {@link net.dv8tion.jda.core.entities.MessageChannel MessageChannel} channel = event.getChannel();
122  *     final long time = System.currentTimeMillis();
123  *    {@literal RestAction<Message>} action = channel.sendMessage("Calculating Response Time...");
124  *     {@link java.util.function.Consumer Consumer}{@literal <Message>} callback = (message) {@literal ->  {
125  *        Message m = message; // ^This is a lambda parameter!^
126  *        m.editMessage("Response Time: " + (System.currentTimeMillis() - time) + "ms").queue();
127  *        // End with queue() to not block the callback thread!
128  *      }};
129  *     // You can also inline this with the queue parameter: action.queue(m {@literal ->} m.editMessage(...).queue());
130  *     action.{@link #queue(Consumer) queue(callback)};
131  * </code></pre>
132  *
133  * <h2>Example Complete:</h2>
134  *
135  * <pre><code>
136  *     {@link net.dv8tion.jda.core.entities.MessageChannel MessageChannel} channel = event.getChannel();
137  *     final long time = System.currentTimeMillis();
138  *    {@literal RestAction<Message>} action = channel.sendMessage("Calculating Response Time...");
139  *     Message message = action.{@link #complete() complete()};
140  *     message.editMessage("Response Time: " + (System.currentTimeMillis() - time) + "ms").queue();
141  *     // End with {@link #queue() queue()} to not block the callback thread!
142  * </code></pre>
143  *
144  * <h2>Example Planning:</h2>
145  *
146  * <pre><code>
147  *     {@link net.dv8tion.jda.core.entities.MessageChannel MessageChannel} channel = event.getChannel();
148  *    {@literal RestAction<Message>} action = channel.sendMessage("This message will destroy itself in 5 seconds!");
149  *     action.queue((message) {@literal ->} message.delete().{@link #queueAfter(long, TimeUnit) queueAfter(5, TimeUnit.SECONDS)});
150  * </code></pre>
151  *
152  * <p><b>Developer Note:</b> It is generally a good practice to use asynchronous logic because
153  * blocking threads requires resources which can be avoided by using callbacks over blocking
154  * operations: <br>
155  * {@link #queue(Consumer)} {@literal >} {@link #complete()}
156  *
157  * <p>There is a dedicated <a href="https://github.com/DV8FromTheWorld/JDA/wiki/7)-Using-RestAction"
158  * target="_blank">wiki page</a> for RestActions that can be useful for learning.
159  *
160  * @param <T> The generic response type for this RestAction
161  * @since 3.0
162  */

173               "RestAction queue returned failure: [{}] {}",

175               t.getMessage());
176         }

311   /**
312    * Submits a Request for execution.
313    *
314    * <p><b>This method is asynchronous</b>
315    *
316    * @param success The success callback that will be called at a convenient time for the API. (can
317    *     be null)
318    * @param failure The failure callback that will be called if the Request encounters an exception
319    *     at its execution point.
320    */
321   public void queue(Consumer<T> success, Consumer<Throwable> failure) {
322     Route.CompiledRoute route = finalizeRoute();
323     Checks.notNull(route, "Route");
324     RequestBody data = finalizeData();
325     CaseInsensitiveMap<String, String> headers = finalizeHeaders();
326     BooleanSupplier finisher = getFinisher();
327     if (success == null) success = DEFAULT_SUCCESS == null ? FALLBACK_CONSUMER : DEFAULT_SUCCESS;
328     if (failure == null) failure = DEFAULT_FAILURE == null ? FALLBACK_CONSUMER : DEFAULT_FAILURE;
329     api.getRequester()
330         .request(
331             new Request<>(this, success, failure, finisher, true, data, rawData, route, headers));
332   }

373   /**
374    * Blocks the current Thread and awaits the completion of an {@link #submit()} request. <br>
375    * Used for synchronous logic.
376    *
377    * <p><b>This might throw {@link java.lang.RuntimeException RuntimeExceptions}</b>
378    *
379    * @return The response value
380    */
381   public T complete() {
382     try {
383       return complete(true);
384     } catch (RateLimitedException ignored) {
385       // This is so beyond impossible, but on the off chance that the laws of nature are rewritten
386       // after the writing of this code, I'm placing this here.
387       // Better safe than sorry?
388       throw new AssertionError(ignored);
389     }
390   }

392   /**
393    * Blocks the current Thread and awaits the completion of an {@link #submit()} request. <br>
394    * Used for synchronous logic.
395    *
396    * @param shouldQueue Whether this should automatically handle rate limitations (default true)
397    * @throws RateLimitedException If we were rate limited and the {@code shouldQueue} is false. Use
398    *     {@link #complete()} to avoid this Exception.
399    * @return The response value
400    */
401   public T complete(boolean shouldQueue) throws RateLimitedException {
402     try {
403       return submit(shouldQueue).get();
404     } catch (Throwable e) {
405       if (e instanceof ExecutionException) {
406         Throwable t = e.getCause();
407         if (t instanceof RateLimitedException) throw (RateLimitedException) t;
408         else if (t instanceof PermissionException) throw (PermissionException) t;
409         else if (t instanceof ErrorResponseException) throw (ErrorResponseException) t;
410       }
411       throw new RuntimeException(e);
412     }
413   }

468     return executor.schedule((Callable<T>) this::complete, delay, unit);

471   /**
472    * Blocks the current Thread for the specified delay and calls {@link #complete()} when delay has
473    * been reached. <br>
474    * If the specified delay is negative this action will execute immediately. (see: {@link
475    * TimeUnit#sleep(long)})
476    *
477    * @param delay The delay after which to execute a call to {@link #complete()}
478    * @param unit The {@link java.util.concurrent.TimeUnit TimeUnit} which should be used (this will
479    *     use {@link java.util.concurrent.TimeUnit#sleep(long) unit.sleep(delay)})
480    * @throws java.lang.IllegalArgumentException If the specified {@link
481    *     java.util.concurrent.TimeUnit TimeUnit} is {@code null}
482    * @throws java.lang.RuntimeException If the sleep operation is interrupted
483    * @return The response value
484    */
485   public T completeAfter(long delay, TimeUnit unit) {
486     Checks.notNull(unit, "TimeUnit");
487     try {
488       unit.sleep(delay);
489       return complete();
490     } catch (InterruptedException e) {
491       throw new RuntimeException(e);
492     }
493   }

523   /**
524    * Schedules a call to {@link #queue(java.util.function.Consumer)} to be executed after the
525    * specified {@code delay}. <br>
526    * This is an <b>asynchronous</b> operation that will return a {@link
527    * java.util.concurrent.ScheduledFuture ScheduledFuture} representing the task.
528    *
529    * <p>This operation gives no access to the failure callback. <br>
530    * Use {@link #queueAfter(long, java.util.concurrent.TimeUnit, java.util.function.Consumer,
531    * java.util.function.Consumer)} to access the failure consumer for {@link
532    * #queue(java.util.function.Consumer, java.util.function.Consumer)}!
533    *
534    * <p>The global JDA {@link java.util.concurrent.ScheduledExecutorService
535    * ScheduledExecutorService} is used for this operation. <br>
536    * You can change the core pool size for this Executor through {@link
537    * net.dv8tion.jda.core.JDABuilder#setCorePoolSize(int) JDABuilder.setCorePoolSize(int)} or
538    * provide your own Executor with {@link #queueAfter(long, java.util.concurrent.TimeUnit,
539    * java.util.function.Consumer, java.util.concurrent.ScheduledExecutorService)}
540    *
541    * @param delay The delay after which this computation should be executed, negative to execute
542    *     immediately
543    * @param unit The {@link java.util.concurrent.TimeUnit TimeUnit} to convert the specified {@code
544    *     delay}
545    * @param success The success {@link java.util.function.Consumer Consumer} that should be called
546    *     once the {@link #queue(java.util.function.Consumer)} operation completes successfully.
547    * @throws java.lang.IllegalArgumentException If the provided TimeUnit is {@code null}
548    * @return {@link java.util.concurrent.ScheduledFuture ScheduledFuture} representing the delayed
549    *     operation

756   /*
757      useful for final permission checks:
758 
759      @Override
760      protected BooleanSupplier finalizeChecks()
761      {
762          // throw exception, if missing perms
763          return () -> hasPermission(Permission.MESSAGE_WRITE);
764      }
765   */
766   protected static class CheckWrapper implements BooleanSupplier {
767     public static final CheckWrapper EMPTY =
768         new CheckWrapper(null, null) {
769           public boolean getAsBoolean() {
770             return true;
771           }
772         };
773 
774     protected final BooleanSupplier pre;
775     protected final BooleanSupplier wrapped;
776 
777     public CheckWrapper(BooleanSupplier wrapped, BooleanSupplier pre) {
778       this.pre = pre;
779       this.wrapped = wrapped;
780     }
781 
782     public boolean pre() {
783       return pre == null || pre.getAsBoolean();
784     }
785 
786     public boolean test() {
787       return wrapped == null || wrapped.getAsBoolean();
788     }
789 
790     @Override
791     public boolean getAsBoolean() {
792       return pre() && test();
793     }
